# Semantic Scholar API integration

## Purpose
Semantic Scholar API integration for academic and scientific research workflows.

## Instructions
1. Define the search query and scope
2. Search using the platform API or interface
3. Filter and rank results by relevance
4. Extract metadata and key information
5. Organize results for further analysis

## Search
- Use advanced search operators for precise queries
- Filter by date range, venue, author, or topic
- Sort by relevance, recency, or citation count
- Save searches for later retrieval
- Set up alerts for new papers matching criteria

## Analysis
- Extract paper metadata (title, authors, abstract, citations)
- Build citation graphs and find related work
- Identify key papers and survey articles
- Track research trends over time
- Cross-reference findings across sources

## Organization
- Export citations in standard formats (BibTeX, RIS)
- Organize papers into collections or projects
- Add notes and annotations to papers
- Tag papers for cross-cutting categorization
- Share collections with collaborators

## Best Practices
- Use multiple search engines for comprehensive coverage
- Verify paper authenticity and peer review status
- Track citation metrics but don't over-rely on them
- Read primary sources rather than depending on summaries
- Keep reference libraries organized and up to date
