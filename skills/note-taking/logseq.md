# Logseq

## Purpose
Use Logseq for outliner-based knowledge management with bidirectional linking and daily journals.

## Instructions
1. Set up Logseq graph directory
2. Use daily journal as primary capture
3. Link concepts across pages
4. Build structured knowledge trees
5. Query and retrieve with datalog

## Core Concepts
- Block-based editing (everything is an outline)
- Bidirectional links with [[page references]]
- Daily journal for chronological capture
- Tags and properties for metadata
- Advanced queries with Datalog

## Workflow
1. Capture in daily journal
2. Tag with relevant page links
3. Process and organize blocks
4. Review linked references
5. Build structured pages from fragments

## Best Practices
- Use daily journals as the default capture point
- Link liberally to build knowledge graph
- Use templates for consistent structures
- Review linked references for serendipitous connections
- Back up the graph directory regularly
