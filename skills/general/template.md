# Template Generation

## Purpose
Generate templates and boilerplate code for common project structures, configurations, and documents.

## Instructions
1. Identify the template type needed
2. Gather project-specific parameters
3. Generate the template with appropriate customization
4. Validate the generated output
5. Provide instructions for further customization

## Template Types
- Project scaffolding (README, gitignore, CI config)
- Configuration files (Docker, K8s, Terraform)
- Code templates (API routes, test files, models)
- Document templates (PRDs, ADRs, runbooks)
- Email and communication templates

## Best Practices
- Keep templates minimal and opinionated
- Include comments explaining configurable sections
- Validate generated output against schema
- Provide examples for complex templates
- Version control templates for consistency
