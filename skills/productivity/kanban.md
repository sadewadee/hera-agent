# Kanban Workflow

## Purpose
Implement Kanban methodology for visual workflow management with work-in-progress limits.

## Instructions
1. Define workflow stages (columns)
2. Set WIP limits per column
3. Visualize work items on the board
4. Pull items through the workflow
5. Measure flow metrics and optimize

## Core Principles
- **Visualize workflow**: Make all work visible on the board
- **Limit WIP**: Cap items per stage to prevent overload
- **Manage flow**: Optimize for smooth, predictable delivery
- **Make policies explicit**: Define done criteria per stage
- **Improve collaboratively**: Regular retrospectives

## Metrics
- Lead time: request to delivery
- Cycle time: start to completion
- Throughput: items completed per period
- WIP: items in progress at any time
- Blocked items: bottleneck indicators

## Best Practices
- Start with your current process (don't reorganize immediately)
- Set WIP limits conservatively (start low, adjust up)
- Focus on finishing items, not starting new ones
- Use swim lanes for different work types
- Review and adjust board regularly
