# CI/CD pipeline development

## Purpose
CI/CD pipeline development following industry best practices for reliable software systems.

## Instructions
1. Assess requirements and constraints
2. Design solution following established patterns
3. Implement with proper testing
4. Review for security and performance
5. Document and deploy

## Core Principles
- Follow established conventions and standards
- Write tests alongside implementation
- Handle errors explicitly
- Log structured data for observability
- Document decisions and trade-offs

## Best Practices
- Use version control with meaningful commits
- Automate testing and deployment
- Monitor production systems
- Keep dependencies updated
- Review code before merging
