# GraphQL API design and implementation

## Purpose
GraphQL API design and implementation for building reliable, maintainable, and scalable software systems.

## Instructions
1. Assess the current architecture and requirements
2. Design the solution following established patterns
3. Implement with proper error handling and testing
4. Review for security, performance, and maintainability
5. Document decisions and deployment procedures

## Design Principles
- SOLID principles for object-oriented design
- KISS: Keep it simple and straightforward
- DRY: Don't repeat yourself (but don't over-abstract)
- YAGNI: You aren't gonna need it
- Separation of concerns

## Implementation Standards
- Write tests before or alongside implementation
- Use type safety when available
- Handle errors explicitly at every level
- Log structured data for observability
- Follow language-specific conventions and idioms

## Code Quality
- Consistent formatting with automated tools
- Meaningful variable and function names
- Functions do one thing well
- Minimize side effects
- Document non-obvious decisions

## Performance
- Measure before optimizing
- Profile to find actual bottlenecks
- Choose appropriate data structures
- Consider caching for repeated computations
- Load test under realistic conditions

## Best Practices
- Use version control with meaningful commits
- Automate testing and deployment
- Review code before merging
- Monitor production systems
- Keep dependencies updated and audited
