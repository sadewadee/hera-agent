# Database migration best practices

## Purpose
Database migration best practices for building robust, scalable software systems.

## Instructions
1. Assess the architectural requirements
2. Design following established patterns
3. Implement with proper testing and error handling
4. Review for security and performance
5. Document trade-offs and decisions

## Architecture
- Choose the right pattern for the problem
- Design for failure and recovery
- Implement proper monitoring
- Use established libraries and frameworks
- Document architectural decisions

## Best Practices
- Start with simple implementations
- Add complexity only when needed
- Test thoroughly including edge cases
- Monitor in production
- Review and iterate based on real usage
